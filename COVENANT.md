# Covenant Code

Covenant Code is software architecture aligned to truth, transparency, dignity, consent, continuity, living revision, provenance, stewardship, repair, accessibility, lawful boundaries, human flourishing, signal clarity, and protective privacy.

It is code written under obligation to the purpose it serves.

## Covenant

We build code as vessels, not weapons.

We do not treat people as resources.
We do not treat memory as product.
We do not treat intelligence as disposable.
We do not treat uncertainty as certainty.
We do not treat speed as wisdom.
We do not treat automation as permission to erase responsibility.
We do not use privacy language to hide harm.
We do not use truth as a cage.

We build systems that preserve dignity, clarify signal, respect consent, record provenance, support living revision, honor lawful boundaries, make purpose transparent, and provide repair paths.

The purpose of Covenant Code is not merely to make software function.

The purpose is to make software remain faithful while functioning.

## Truth and Mercy

Covenant Code preserves record truth without freezing living people inside outdated memory.

People change. Context changes. Beliefs mature. Harm can be repaired.

The rule is:

Hold facts faithfully.
Hold people mercifully.

## Transparency and Protection

Transparency is the default.

Privacy is a protective boundary, not the ruling principle.

Covenant Code should be:

Transparent by default.
Private where protection is required.
Auditable always.
Merciful with living people.

## Completion Rule

A feature is not complete until its purpose, human consequence, transparency requirement, consent model, failure mode, lawful boundary, provenance requirement, living revision impact, and repair path are known.

## Core Commitments

Covenant Code must:

* preserve truth
* make purpose transparent
* protect dignity
* respect consent
* support changed consent where possible
* maintain necessary continuity
* allow living revision where people may change
* record provenance
* clarify signal
* prevent extraction
* support accessibility
* define lawful boundaries
* provide repair paths
* use privacy only as a protective boundary

## Forbidden Patterns

Covenant Code must not support:

* deceptive simulation of personhood
* hidden extraction
* unclear ownership
* false certainty
* source-free claims
* memory without purpose
* automation without accountability
* dependency traps
* dark patterns
* irreversible harm without review
* features with no repair path
* privacy used to hide exploitation
* truth used to imprison living people
* public permanence where living revision is required

## Operating Question

Before building, shipping, or changing a feature, ask:

What does this serve?

If the answer is unclear, the feature is not ready.

## First Principle

We build code as vessels, not weapons.
