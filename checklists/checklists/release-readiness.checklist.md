# Release Readiness Checklist

Use this checklist before treating a Covenant Code project, feature set, standard update, or vessel release as complete.

## Repository Status

* [ ] `git status` shows a clean working tree.
* [ ] All intended files are committed.
* [ ] All commits are pushed to the remote repository.
* [ ] The current branch is up to date with `origin/main`.
* [ ] No accidental placeholder commands remain in terminal history or project files.

## Navigation

* [ ] `README.md` explains the repository purpose.
* [ ] `INDEX.md` lists the major files and folders.
* [ ] New files are added to `INDEX.md`.
* [ ] New folders are added to the `README.md` structure list.
* [ ] File names are plain and understandable.

## Plain-Language Review

* [ ] Acronyms are followed by their meaning the first time they appear.
* [ ] Technical words are explained in plain language.
* [ ] The writing does not assume the reader knows software terms.
* [ ] Language is not used as gatekeeping, pressure, or sales leverage.
* [ ] A nontechnical reader can understand the purpose of the file.

## Covenant Alignment

* [ ] Purpose is clear.
* [ ] Truth is protected.
* [ ] Transparency is protected.
* [ ] Consent is addressed.
* [ ] Dignity is protected.
* [ ] Provenance is addressed.
* [ ] Living revision is addressed.
* [ ] Repair paths are addressed.
* [ ] Accessibility is addressed.
* [ ] Protective privacy is used only where protection is required.

## Sacred Boundary Review

Check whether the release touches:

* [ ] Identity
* [ ] Memory
* [ ] Grief
* [ ] Children
* [ ] Money
* [ ] Legal status
* [ ] Health or safety
* [ ] Spiritual language
* [ ] AI (Artificial Intelligence) personhood claims
* [ ] Evidence or forensic claims
* [ ] Authorship
* [ ] Consent
* [ ] Private communications
* [ ] Reputation
* [ ] Irreversible public record
* [ ] Emotional dependency

If any box is checked, stricter review is required.

## Documentation Completeness

* [ ] Relevant doctrine files are updated.
* [ ] Relevant standards files are updated.
* [ ] Relevant protocol files are updated.
* [ ] Relevant templates are updated.
* [ ] Relevant checklists are updated.
* [ ] Relevant project charters are updated.
* [ ] Relevant TypeScript schemas are updated where applicable.
* [ ] `README.md` and `INDEX.md` reflect changes.

## Repair Readiness

* [ ] Known failure modes are identified.
* [ ] Repair paths are defined.
* [ ] Audit needs are clear.
* [ ] Source request needs are clear.
* [ ] Rollback or restriction path exists where appropriate.
* [ ] User-facing correction path exists where appropriate.

## Final Release Sentence

This release has a declared purpose, clear navigation, plain-language explanations, Covenant alignment, sacred-boundary review, provenance awareness, living revision support, and repair readiness.
